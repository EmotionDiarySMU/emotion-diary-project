# 프로젝트 개요 (Project Overview)

## 1. 프로젝트 소개

### 1.1 프로젝트명
**감정 일기장 (Emotion Diary)**

### 1.2 프로젝트 목적
사용자가 일상의 감정과 스트레스 수치를 기록하고, 이를 시각화된 통계 그래프로 확인할 수 있는 감정 관리 애플리케이션입니다. 사용자는 자신의 감정 패턴을 분석하여 정신 건강 관리에 도움을 받을 수 있습니다.

### 1.3 주요 기능
1. **사용자 인증**
   - 로그인/회원가입 기능
   - 개인별 일기 데이터 관리

2. **일기 작성**
   - 제목, 내용 입력
   - 12가지 감정 중 최대 4개 선택
   - 각 감정별 수치 입력 (0-100)
   - 스트레스 수치 입력 (0-100)

3. **일기 열람 및 관리**
   - 제목별 검색
   - 날짜별 검색
   - 오름차순/내림차순 정렬
   - 일기 수정 기능

4. **통계 분석**
   - 감정별 통계 그래프 (12가지 감정)
   - 스트레스 지수 그래프
   - 주간/월간/연간 기간 선택
   - 평균 스트레스 지수 표시

## 2. 기술 스택

### 2.1 개발 환경
- **언어**: Java 17
- **빌드 도구**: Maven
- **IDE**: IntelliJ IDEA
- **데이터베이스**: MySQL 8.0.33
- **인코딩**: UTF-8

### 2.2 프레임워크 및 라이브러리
- **프론트엔드**: Java Swing
- **백엔드**: Maven 기반 Java 프로젝트
- **차트 라이브러리**: JFreeChart 1.5.3
- **DB 연동**: MySQL Connector/J 8.0.33

### 2.3 아키텍처 패턴
- **MVC (Model-View-Controller) 패턴** 준수
  - Model: DAO 클래스 (데이터베이스 접근)
  - View: Swing 기반 GUI 클래스
  - Controller: 비즈니스 로직 처리 클래스

## 3. 감정 정의

### 3.1 긍정적인 감정 (6가지)
| 감정명 | 이모지 | 설명 |
|--------|--------|------|
| 행복 | 😊 | 가장 기본적이고 따뜻한 미소 |
| 신남 | 😆 | 파티 모자를 쓰고 신난 얼굴 |
| 설렘 | 😍 | 사랑스럽고 설레는 느낌 |
| 편안 | 😌 | 안도하고 편안한 얼굴 |
| 재미 | 😂 | 재미있어서 눈물 나게 웃는 얼굴 |
| 고마움 | 🤗 | 양손을 흔들며 미소 짓는 얼굴 |

### 3.2 부정적인 감정 (6가지)
| 감정명 | 이모지 | 설명 |
|--------|--------|------|
| 슬픔 | 😢 | 눈물이 흐르는 슬픈 얼굴 |
| 분노 | 😠 | 화가 난 얼굴 |
| 불안 | 😰 | 식은땀을 흘리며 불안한 얼굴 |
| 민망 | 😅 | 멋쩍게 웃으며 땀 흘리는 얼굴 |
| 당황 | 😧 | 당황한 얼굴 |
| 미안함 | 😔 | 근심이 많아보이는 얼굴 |

### 3.3 감정 입력 규칙
- 일기 작성 시 **최대 4개**의 감정 선택 가능
- 선택된 각 감정마다 **0~100 사이의 수치** 입력
- 스트레스 수치도 **0~100 사이의 값** 입력

## 4. 프로젝트 구조

### 4.1 패키지 구조
```
com.diary.emotion/
├── model/          # 데이터 모델 및 DAO
├── view/           # Swing GUI 컴포넌트
├── controller/     # 비즈니스 로직
└── util/           # 유틸리티 클래스
```

### 4.2 주요 클래스
- **AppLauncher**: 애플리케이션 진입점 (main 메소드)
- **MainApplication**: 메인 화면 컨테이너 (JPanel)
- **DatabaseUtil**: 데이터베이스 초기화 및 연결 관리
- **StatisticsView**: 통계 화면 UI
- **StatisticsController**: 통계 로직 처리
- **StatisticsDAO**: 통계 데이터 조회

## 5. 데이터베이스 설계

### 5.1 데이터베이스명
- **emotion_diary**
- 문자셋: UTF8MB4 (이모지 저장 지원)
- Collation: utf8mb4_unicode_ci (대소문자 구분 없음)

### 5.2 주요 테이블
1. **user**: 사용자 정보
2. **diary**: 일기 내용 및 스트레스 수치
3. **emotion**: 감정별 수치 정보
4. **question**: 질문 데이터 (추후 확장용)

*자세한 스키마는 `05_DATABASE_SCHEMA.md` 참조*

## 6. UI/UX 설계

### 6.1 화면 구성
1. **로그인/회원가입 화면** (최초 실행)
   - 아이디, 비밀번호 입력
   - 로그인 버튼
   - 회원가입 버튼

2. **메인 화면** (로그인 후)
   - 상단 네비게이션: 일기 쓰기 / 열람 / 통계
   - 중앙 컨텐츠 영역: 선택된 탭의 내용 표시

3. **일기 쓰기 탭**
   - 제목, 내용 입력
   - 감정 선택 (최대 4개)
   - 감정별 수치 입력
   - 스트레스 수치 입력

4. **열람 탭**
   - 검색 기능 (제목/날짜)
   - 정렬 기능 (오름차순/내림차순)
   - 일기 목록
   - 수정 기능

5. **통계 탭**
   - 기간 선택 (주간/월간/연간)
   - 감정 통계 그래프 (막대 차트)
   - 스트레스 지수 그래프 (꺾은선 차트)
   - 평균 스트레스 지수 라벨

### 6.2 디자인 가이드
- **컬러 팔레트**
  - 메인 배경: 파스텔 블루 (RGB: 230, 240, 255)
  - 네비게이션: 파스텔 옐로우 (RGB: 255, 255, 220)
- **폰트**
  - 기본: SansSerif
  - 제목: Bold 16pt
  - 본문: Plain 11pt

### 6.3 UI 참고 자료
