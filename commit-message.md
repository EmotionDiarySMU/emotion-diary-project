docs: README.md를 표준 규격에 맞게 전면 개선

프로젝트의 README.md를 업계 표준 규격과 모범 사례에 따라 대폭 개선하여
새로운 사용자와 기여자들이 프로젝트를 쉽게 이해하고 시작할 수 있도록 함.

## 주요 변경사항

### 추가된 섹션
- 기술 스택 배지 (Java 21, MySQL 8.0, Maven, Swing, JFreeChart)
- 상세한 목차 (전체 섹션 네비게이션)
- 프로젝트 소개 확장 (개발 목적 및 핵심 가치 테이블)
- 시스템 아키텍처 다이어그램 (3-Tier: Presentation → Controller → DAO → DB)
- 데이터베이스 스키마 (ERD 다이어그램 + 4개 테이블 상세 스펙)
- 스크린샷 섹션 (추후 업데이트 예정 표시)
- 기여 가이드 (Fork → Branch → Commit → PR 흐름)
- 커밋 컨벤션 가이드 (feat, fix, docs, style, refactor 등)
- 코드 스타일 가이드

### 개선된 섹션
- 주요 기능을 5개 카테고리로 재구성 및 상세화
  * 사용자 인증 시스템 (회원가입/로그인/로그아웃/계정삭제)
  * 일기 작성 기능 (12개 이모지, 50개 랜덤 질문, 스트레스 슬라이더)
  * 일기 조회 및 검색 (제목/날짜 검색, 정렬, 다중 창 지원)
  * 일기 수정 및 삭제 (수정 모드, 취소, 이중 확인 삭제)
  * 통계 분석 (주간/월간/연간 MVC 패턴)

- 기술 스택을 표 형식으로 재구성 (구분/기술/버전/용도)
- 프로젝트 구조를 이모지와 함께 상세 설명 추가
- 설치 및 실행 방법을 Step-by-Step 가이드로 개선
  * DB 연결 설정 가이드 추가
  * 3가지 실행 방법 제시 (Maven/IDE/JAR)
- 사용 방법을 5단계 플로우로 재구성

### 추가된 상세 정보
- 데이터베이스 테이블 구조 (user, diary, emotion, question)
- 각 테이블의 컬럼명, 타입, 제약조건, 설명
- 의존성 라이브러리 버전 및 용도
- 시스템 아키텍처의 계층별 역할 설명
- MVC 패턴을 활용한 통계 모듈 설계

## 문서 구조

```
1. 프로젝트 헤더 (제목, 설명, 배지)
2. 목차
3. 프로젝트 소개 (개발 목적, 핵심 가치)
4. 주요 기능 (5개 카테고리)
5. 시스템 아키텍처 (3-Tier 다이어그램)
6. 기술 스택 (표 형식 + 의존성 코드)
7. 프로젝트 구조 (파일/폴더 트리 + 설명)
8. 데이터베이스 스키마 (ERD + 테이블 상세)
9. 설치 및 실행 방법 (4단계 가이드)
10. 사용 방법 (5단계 플로우)
11. 스크린샷 (예정)
12. 기여 가이드 (방법 + 컨벤션 + 스타일)
13. 개발팀 (연락처)
14. 라이선스
15. 푸터 (감성 메시지)
```

## 기대 효과

- 새로운 사용자가 프로젝트를 쉽게 이해하고 설치 가능
- 기여자가 코드 구조와 설계 원칙을 빠르게 파악 가능
- 프로젝트의 전문성과 완성도 향상
- GitHub에서의 가시성 및 접근성 개선
