## 1. 프로젝트 개요

### 1.1 프로젝트 목적
사용자가 일상의 감정과 스트레스를 체계적으로 기록하고, 시각화된 통계 그래프를 통해 자신의 감정 패턴을 분석하여 정신 건강 관리에 도움을 받을 수 있는 데스크톱 애플리케이션 개발

### 1.2 대상 사용자
- 자신의 감정을 기록하고 관리하고 싶은 일반 사용자
- 스트레스 관리가 필요한 사용자
- 감정 패턴 분석을 통해 자기 이해를 높이고 싶은 사용자
- 일기 작성을 통한 정서적 안정을 원하는 사용자

### 1.3 핵심 가치
- **감정 인식**: 세분화된 감정 분류로 명확한 감정 기록
- **자기 성찰**: 시간에 따른 감정 패턴 시각화
- **정서적 지원**: 일기 작성을 통한 감정 정리
- **데이터 기반 분석**: 통계를 통한 장기적 감정 변화 추적

## 2. 기능적 요구사항

### 2.1 사용자 인증 (FR-01)

#### FR-01-01: 회원가입
- **설명**: 신규 사용자가 계정을 생성할 수 있어야 함
- **입력**: 사용자 ID, 비밀번호, 비밀번호 확인
- **처리 규칙**:
  - 사용자 ID는 중복될 수 없음
  - 비밀번호와 비밀번호 확인이 일치해야 함
  - 사용자 ID는 최대 20자까지 입력 가능
  - 비밀번호는 최대 20자까지 입력 가능
- **출력**: 회원가입 성공 시 로그인 화면으로 이동
- **예외 처리**:
  - 중복된 ID 입력 시 경고 메시지 표시
  - 비밀번호 불일치 시 경고 메시지 표시
  - 빈 필드 존재 시 입력 요청 메시지 표시

#### FR-01-02: 로그인
- **설명**: 기존 사용자가 계정으로 로그인할 수 있어야 함
- **입력**: 사용자 ID, 비밀번호
- **처리 규칙**:
  - 데이터베이스의 사용자 정보와 일치해야 함
  - 로그인 성공 시 세션에 사용자 ID 저장
- **출력**: 로그인 성공 시 메인 화면으로 이동
- **예외 처리**:
  - 로그인 실패 시 "아이디 또는 비밀번호가 일치하지 않습니다" 메시지 표시
  - 빈 필드 존재 시 입력 요청 메시지 표시

#### FR-01-03: 세션 관리
- **설명**: 로그인 상태를 유지하고 관리해야 함
- **기능**:
  - 현재 로그인한 사용자 ID 저장
  - 로그인 상태 확인
  - 로그아웃 기능
- **처리 규칙**:
  - 모든 개인 데이터 조회는 현재 로그인한 사용자의 것만 표시
  - 로그아웃 시 세션 정보 초기화 및 로그인 화면으로 이동

### 2.2 일기 작성 (FR-02)

#### FR-02-01: 일기 기본 정보 입력
- **설명**: 사용자가 일기의 제목과 내용을 작성할 수 있어야 함
- **입력**: 제목 (최대 50자), 내용 (TEXT)
- **처리 규칙**:
  - 제목은 필수 입력 사항
  - 내용은 선택 사항
  - 제목이 비어있으면 저장 불가
- **출력**: 입력된 제목과 내용이 화면에 표시
- **예외 처리**:
  - 제목 미입력 시 "제목을 입력해주세요" 경고 메시지

#### FR-02-02: 감정 선택 및 수치 입력
- **설명**: 사용자가 12가지 감정 중 최대 4개를 선택하고 각 감정의 강도를 입력할 수 있어야 함
- **입력**: 
  - 감정 선택 (체크박스, 최대 4개)
  - 각 감정별 수치 (슬라이더, 0-100 범위)
- **감정 목록**:
  - **긍정 감정 (6개)**: 😊 행복, 😆 신남, 😍 설렘, 😌 편안, 😂 재미, 🤗 고마움
  - **부정 감정 (6개)**: 😢 슬픔, 😠 분노, 😰 불안, 😅 민망, 😧 당황, 😔 미안함
- **처리 규칙**:
  - 최소 1개, 최대 4개의 감정 선택 가능
  - 4개 초과 선택 시 경고 메시지 표시 및 추가 선택 불가
  - 선택된 감정만 수치 슬라이더 활성화
  - 감정 수치는 0-100 사이의 정수값
- **출력**: 선택된 감정과 수치가 화면에 표시
- **예외 처리**:
  - 감정을 하나도 선택하지 않은 경우 "최소 1개의 감정을 선택해주세요" 경고

#### FR-02-03: 스트레스 수치 입력
- **설명**: 사용자가 현재 스트레스 수준을 입력할 수 있어야 함
- **입력**: 스트레스 수치 (슬라이더, 0-100 범위)
- **처리 규칙**:
  - 기본값: 50
  - 0-100 사이의 정수값
- **출력**: 선택된 스트레스 수치 표시

#### FR-02-04: 일기 저장
- **설명**: 작성된 일기와 감정 정보를 데이터베이스에 저장해야 함
- **처리 규칙**:
  - 일기 저장 시 현재 로그인한 사용자 ID와 작성 일시 자동 기록
  - 트랜잭션 처리로 일기와 감정 데이터의 원자성 보장
  - 일기 저장 실패 시 감정 데이터도 롤백
- **출력**: 저장 성공 시 "일기가 저장되었습니다" 메시지 및 목록 화면으로 이동
- **예외 처리**:
  - 데이터베이스 오류 시 "저장에 실패했습니다" 메시지

### 2.3 일기 열람 및 관리 (FR-03)

#### FR-03-01: 일기 목록 조회
- **설명**: 현재 사용자의 모든 일기를 목록으로 조회할 수 있어야 함
- **표시 정보**: 번호, 날짜, 제목, 스트레스 수치, 감정 이모지
- **처리 규칙**:
  - 현재 로그인한 사용자의 일기만 표시
  - 테이블 형태로 표시
  - 기본 정렬: 최신순 (작성일 기준 내림차순)
- **출력**: 일기 목록 테이블

#### FR-03-02: 일기 검색
- **설명**: 제목 키워드로 일기를 검색할 수 있어야 함
- **입력**: 검색 키워드
- **처리 규칙**:
  - 제목에 검색 키워드가 포함된 일기만 표시
  - 대소문자 구분 없음
  - 빈 검색어 입력 시 전체 목록 표시
- **출력**: 검색 결과 목록
- **추가 기능**: "전체보기" 버튼으로 검색 초기화

#### FR-03-03: 일기 정렬
- **설명**: 일기 목록을 날짜 기준으로 정렬할 수 있어야 함
- **정렬 옵션**:
  - 최신순 (작성일 내림차순)
  - 오래된순 (작성일 오름차순)
- **처리 규칙**:
  - 콤보박스로 정렬 옵션 선택
  - 선택 즉시 목록 재정렬
- **출력**: 정렬된 일기 목록

#### FR-03-04: 일기 상세보기
- **설명**: 선택한 일기의 전체 내용을 확인할 수 있어야 함
- **표시 정보**: 
  - 작성일
  - 제목
  - 내용
  - 감정 (이모지 + 수치)
  - 스트레스 수치
- **처리 규칙**:
  - 목록에서 일기 선택 후 "상세보기" 버튼 클릭
  - 새 창 (JDialog)으로 표시
- **출력**: 일기 상세 정보 다이얼로그
- **예외 처리**:
  - 일기 미선택 시 "일기를 선택해주세요" 경고 메시지

#### FR-03-05: 일기 수정
- **설명**: 기존 일기를 수정할 수 있어야 함
- **수정 가능 항목**: 제목, 내용, 감정, 감정 수치, 스트레스 수치
- **처리 규칙**:
  - 목록에서 일기 선택 후 "수정" 버튼 클릭
  - 수정 화면으로 전환 (기존 데이터 로드)
  - 트랜잭션 처리로 데이터 무결성 보장
  - 기존 감정 삭제 후 새 감정 데이터 저장
- **출력**: 수정 성공 시 "일기가 수정되었습니다" 메시지
- **예외 처리**:
  - 일기 미선택 시 "일기를 선택해주세요" 경고 메시지
  - 수정 실패 시 "수정에 실패했습니다" 메시지

#### FR-03-06: 일기 삭제
- **설명**: 일기를 삭제할 수 있어야 함
- **처리 규칙**:
  - 목록에서 일기 선택 후 "삭제" 버튼 클릭
  - 확인 대화상자 표시 ("정말로 삭제하시겠습니까?")
  - 확인 시 일기 및 관련 감정 데이터 삭제 (CASCADE)
- **출력**: 삭제 성공 시 목록에서 제거 및 목록 갱신
- **예외 처리**:
  - 일기 미선택 시 "일기를 선택해주세요" 경고 메시지
  - 삭제 실패 시 "삭제에 실패했습니다" 메시지

### 2.4 통계 분석 (FR-04)

#### FR-04-01: 감정 통계 차트
- **설명**: 선택한 기간 동안의 감정별 평균 수치를 막대 차트로 표시해야 함
- **차트 유형**: 수평 막대 차트 (Horizontal Bar Chart)
- **표시 정보**:
  - X축: 평균 감정 수치 (0-100)
  - Y축: 감정 이름 (12가지)
  - 긍정 감정: 하늘색 막대
  - 부정 감정: 분홍색 막대
- **처리 규칙**:
  - 선택한 기간 내 각 감정의 평균 수치 계산
  - 기록이 없는 감정은 0으로 표시
  - 기간 변경 시 자동 갱신
- **출력**: JFreeChart 기반 감정 통계 막대 차트

#### FR-04-02: 스트레스 추이 그래프
- **설명**: 선택한 기간 동안의 스트레스 수치 변화를 꺾은선 그래프로 표시해야 함
- **차트 유형**: 꺾은선 차트 (Line Chart)
- **표시 정보**:
  - X축: 날짜/기간
  - Y축: 스트레스 수치 (0-100)
- **처리 규칙**:
  - **주간 모드**: 일별 스트레스 데이터 (7일)
  - **월간 모드**: 일별 스트레스 데이터 (해당 월의 모든 날짜)
  - **연간 모드**: 월별 평균 스트레스 데이터 (12개월)
  - 데이터가 없는 날짜/월은 0으로 표시
  - 기간 변경 시 자동 갱신
- **출력**: JFreeChart 기반 스트레스 추이 꺾은선 차트

#### FR-04-03: 평균 스트레스 지수 표시
- **설명**: 선택한 기간의 평균 스트레스 수치를 숫자로 표시해야 함
- **처리 규칙**:
  - 선택한 기간 내 모든 일기의 스트레스 평균 계산
  - 소수점 첫째 자리까지 표시
  - 데이터가 없으면 0.0 표시
- **출력**: "평균 스트레스 지수: XX.X" 형태의 레이블

#### FR-04-04: 기간 선택
- **설명**: 통계를 확인할 기간을 선택할 수 있어야 함
- **선택 옵션**:
  - **주간 (Week)**: 선택한 날짜 기준 7일 (해당 주의 일요일 ~ 토요일)
  - **월간 (Month)**: 선택한 월의 1일 ~ 말일
  - **연간 (Year)**: 선택한 연도의 1월 ~ 12월
- **입력 방법**:
  - 모드 선택 콤보박스 (주간/월간/연간)
  - 연도 선택 콤보박스
  - 월/주 선택 콤보박스 (모드에 따라 동적 변경)
- **처리 규칙**:
  - 콤보박스 변경 시 즉시 차트 갱신
  - 기본 모드: 주간
  - 기본 기간: 현재 주
- **출력**: 선택된 기간의 통계 차트 및 평균 스트레스 지수

## 3. 비기능적 요구사항

### 3.1 성능 요구사항 (NFR-01)
- **NFR-01-01**: 일기 목록 조회는 1초 이내에 완료되어야 함
- **NFR-01-02**: 통계 차트 생성은 2초 이내에 완료되어야 함
- **NFR-01-03**: 데이터베이스 쿼리 응답 시간은 500ms 이내여야 함
- **NFR-01-04**: UI 전환은 즉각적이어야 함 (체감 지연 없음)

### 3.2 사용성 요구사항 (NFR-02)
- **NFR-02-01**: 모든 기능은 3클릭 이내에 접근 가능해야 함
- **NFR-02-02**: 에러 메시지는 사용자가 이해하기 쉬운 한국어로 표시되어야 함
- **NFR-02-03**: 중요한 작업 (삭제 등)은 확인 대화상자를 통해 실수를 방지해야 함
- **NFR-02-04**: 슬라이더는 현재 값을 실시간으로 표시해야 함
- **NFR-02-05**: 버튼은 명확한 레이블을 가져야 함

### 3.3 신뢰성 요구사항 (NFR-03)
- **NFR-03-01**: 데이터베이스 트랜잭션 실패 시 롤백 처리되어야 함
- **NFR-03-02**: 애플리케이션 비정상 종료 시에도 이미 저장된 데이터는 보존되어야 함
- **NFR-03-03**: 데이터베이스 연결 실패 시 사용자에게 명확한 오류 메시지를 제공해야 함
- **NFR-03-04**: 일기 저장 실패 시 사용자 입력 데이터가 손실되지 않아야 함

### 3.4 보안 요구사항 (NFR-04)
- **NFR-04-01**: 사용자는 자신의 일기만 조회/수정/삭제할 수 있어야 함

## 5. 데이터 요구사항

### 5.1 사용자 데이터 (DR-01)
- **테이블**: `user`
- **필드**:
  - `user_id` (VARCHAR(20), PRIMARY KEY): 사용자 아이디
  - `user_pw` (VARCHAR(20), NOT NULL): 비밀번호
- **제약조건**:
  - `user_id`는 고유해야 함
  - 모든 필드는 NULL 불가

### 5.2 일기 데이터 (DR-02)
- **테이블**: `diary`
- **필드**:
  - `entry_id` (INTEGER, AUTO_INCREMENT, PRIMARY KEY): 일기 고유 ID
  - `user_id` (VARCHAR(20), NOT NULL, FOREIGN KEY): 작성자 ID
  - `title` (VARCHAR(50), NOT NULL): 일기 제목
  - `content` (TEXT): 일기 본문
  - `stress_level` (INTEGER, NOT NULL): 스트레스 수치 (0-100)
  - `entry_date` (DATETIME, NOT NULL): 작성 일시
- **제약조건**:
  - `user_id`는 `user` 테이블의 `user_id` 참조
  - `stress_level`은 0-100 범위
  - `entry_date`는 자동 기록

### 5.3 감정 데이터 (DR-03)
- **테이블**: `emotion`
- **필드**:
  - `emotion_id` (INTEGER, AUTO_INCREMENT, PRIMARY KEY): 감정 레코드 ID
  - `entry_id` (INTEGER, NOT NULL, FOREIGN KEY): 일기 ID
  - `emotion_level` (INTEGER, NOT NULL): 감정 수치 (0-100)
  - `emoji_icon` (VARCHAR(10), NOT NULL): 감정 이모지
- **제약조건**:
  - `entry_id`는 `diary` 테이블의 `entry_id` 참조
  - `emotion_level`은 0-100 범위
  - 하나의 일기당 최대 4개의 감정 레코드

### 5.4 감정 매핑 데이터 (DR-04)
- **테이블**: `emotion_mapping`
- **필드**:
  - `emotion_id` (INTEGER, AUTO_INCREMENT, PRIMARY KEY): 감정 ID
  - `emotion_name` (VARCHAR(20), NOT NULL): 감정 이름
  - `emoji_icon` (VARCHAR(10), NOT NULL): 이모지
  - `emotion_type` (VARCHAR(10), NOT NULL): 감정 유형 (긍정/부정)
- **데이터**: 12개의 감정 정의
  - 긍정: 행복, 신남, 설렘, 편안, 재미, 고마움
  - 부정: 슬픔, 분노, 불안, 민망, 당황, 미안함
